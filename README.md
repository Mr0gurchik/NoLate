# Умный адаптивный будильник NoLate

**Проект представляет собой архитектуру и прототип мобильного приложения на платформе .NET MAUI, предназначенного для динамического расчета времени пробуждения с учетом дорожного трафика и расписания общественного транспорта.**  
Система обеспечивает фоновый мониторинг маршрута пользователя и автоматически сдвигает время срабатывания системного будильника, гарантируя прибытие в пункт назначения точно в срок без участия человека.

---

## О проекте

В условиях современного ритма жизни в мегаполисах стандартные статичные будильники теряют свою эффективность: они не способны учитывать внезапные пробки или изменения в расписании транспорта, что является причиной почти трети всех опозданий на работу. 

Данный проект предлагает **гибридный подход к маршрутизации**: для расчета маршрутов на общественном транспорте и пешком используется собственный развернутый сервер OpenTripPlanner (с доступом по GraphQL, что исключает лимиты на запросы), а для автомобильных маршрутов применяется TomTom API (REST) со встроенным алгоритмическим буфером (+40% ко времени) для адаптации под реалии российских дорог. Это позволяет достичь высокой точности тайм-менеджмента при нулевых затратах на API-инфраструктуру.

## Цель и задачи

**Цель:** спроектировать и разработать кроссплатформенное мобильное приложение «NoLate» на платформе .NET MAUI с реализацией нативных фоновых сервисов Android и интеграцией сторонних картографических API.

**Задачи:**
- проанализировать существующие API для построения маршрутов (OpenTripPlanner, TomTom, Яндекс);
- спроектировать архитектуру локальной базы данных (SQLite) для хранения параметров будильников;
- разработать пользовательский интерфейс (XAML) на .NET MAUI;
- реализовать клиент-серверное взаимодействие через GraphQL (для OTP) и REST (для TomTom);
- разработать нативный фоновый сервис для Android (Foreground Service), работающий в обход спящего режима (Doze Mode);
- реализовать динамический интервал пулинга (опроса серверов) для экономии заряда батареи;
- провести тестирование точности геолокации и алгоритмов динамического сдвига времени будильника.

## Функциональные возможности

- **Динамический пересчет времени** – регулярная фоновая проверка времени в пути. Если оно меняется более чем на 5 минут, будильник автоматически переносится.
- **Учет времени на сборы** – пользователь задает время прибытия и время на утреннюю рутину, а приложение само высчитывает, когда нужно проснуться.
- **Мультимодальная маршрутизация** – поддержка построения маршрутов для автомобилей (с учетом пробок), общественного транспорта и пешеходов.
- **Интерактивная карта** – удобный выбор точки назначения прямо на карте (`MapPage`).
- **Надежное срабатывание** – использование системного `AlarmManager` (Android) и собственных аудиоканалов (`AudioUsageKind.Alarm`) для гарантии пробуждения даже в беззвучном режиме телефона.
- **Локальное хранение** – все настройки и координаты надежно сохраняются в локальной БД устройства.

## Технологии

- **Платформа разработки:** .NET MAUI (C#, XAML)
- **Целевая ОС:** Android (с перспективой портирования на iOS)
- **Локальная База Данных:** SQLite (sqlite-net-pcl)
- **Сетевое взаимодействие:** HttpClient, JSON, GraphQL, REST
- **Маршрутизация (Авто):** TomTom Routing API
- **Маршрутизация (Транспорт):** OpenTripPlanner (self-hosted на удаленном VPS, доступ через туннель zrok)
- **Нативные API Android:** Foreground Services, AlarmManager, BroadcastReceiver, NotificationManager, MediaPlayer

## Архитектура системы (кратко)

Система построена по многослойной модульной схеме:

1. **UI-уровень (Views)** – кроссплатформенный интерфейс пользователя (`MainPage`, `AlarmEditPage`, `MapPage`), написанный на XAML.
2. **Сервисный уровень (Services)** – ядро бизнес-логики: `OTPRouteService` (работа с внешними API), `DatabaseService` (ORM для SQLite), `LocationHelper` (гибридное получение GPS координат).
3. **Платформенный уровень (Android)** – изолированные нативные компоненты: `TrafficService` (фоновый мониторинг маршрутов) и `AlarmReceiver` (обработчик системных прерываний таймера).

Данные о маршрутах запрашиваются асинхронно в фоновом режиме. Система использует динамический интервал опроса (от 1 часа до 10 минут в зависимости от близости времени выхода), чтобы минимизировать нагрузку на сеть и батарею смартфона.

## Установка и запуск (для разработчиков)

1. Склонировать репозиторий проекта на локальный компьютер.
2. Открыть решение `NoLate.sln` в Visual Studio 2022 (требуется установленный workload ".NET Multi-platform App UI development").
3. Восстановить NuGet-пакеты.
4. Проверить актуальность API-ключей в классе `OTPRouteService.cs` (ключ TomTom и URL для zrok туннеля).
5. Выбрать целевой платформой эмулятор Android (API 24+) или физическое Android-устройство.
6. Выполнить сборку и развертывание проекта (F5).
7. При желании можите собрать App файл

> **Примечание:** При первом запуске на устройстве необходимо выдать приложению разрешения на доступ к геолокации, отправку уведомлений и установку точных будильников.
